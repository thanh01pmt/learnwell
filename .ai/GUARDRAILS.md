# LearnWell AI Agent Guardrails

## ❌ Do NOT

1. **Modify shared types** (`src/types/`) without creating an OpenSpec proposal first
2. **Delete or rename** existing translation keys — other languages depend on them
3. **Change database schema** (`packages/db/prisma/schema.prisma`) without a migration file
4. **Remove `AppLayout` wrapper** from pages — sidebar and header must always be visible
5. **Hardcode strings** in components — always use translation keys via `useTranslation()`
6. **Add new routes** without registering them in `featureRegistry.ts`
7. **Import across app boundaries** — `apps/web` must not import from `apps/api` directly; use `packages/` for shared code
8. **Modify `pnpm-lock.yaml`** manually — it is auto-generated by `pnpm install`
9. **Create components outside** `src/components/` — all shared components go there
10. **Use inline styles** — always use TailwindCSS utility classes or `cn()` helper

## ✅ Always

1. **Run `pnpm typecheck`** before committing (catches cross-package type errors)
2. **Add translations** to BOTH `vi/` and `en/` simultaneously
3. **Wrap new pages** in `<AppLayout>` for consistent layout
4. **Use OpenSpec workflow** for architectural changes: `proposal.md` → review → implement
5. **Test with Role Switcher** after any role-related change
6. **Use `@/` import alias** instead of relative paths
7. **Check `openspec/changes/`** before starting work to avoid conflicts with others
8. **Keep PRs small** — under 300 lines changed when possible
9. **Follow Conventional Commits** — `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`
10. **Validate specs** after changes: `openspec validate --strict`

## ⚠️ Be Careful With

1. **`featureRegistry.ts`**: Central registry mapping routes to features — changes affect all role-based access
2. **`RoleContext.tsx`**: Core role management — changes affect every component using `useRole()`
3. **`AppSidebar.tsx`**: Navigation for all roles — adding/removing items affects everyone
4. **`App.tsx`**: Route definitions — ensure new routes are wrapped in appropriate guards
5. **`database.ts`**: Type definitions must stay in sync with Prisma schema (future)

## 🔍 Pre-Flight Checklist

Before submitting any significant change:

- [ ] `pnpm typecheck` passes
- [ ] Both `vi` and `en` translations are in sync
- [ ] New pages wrapped in `AppLayout`
- [ ] New routes registered in `featureRegistry.ts`
- [ ] OpenSpec proposal exists for architectural changes
- [ ] Tested with at least 2 roles (student + teacher)
